# Troubleshooting Guide

This guide provides systematic solutions for the most common issues encountered with the BARS EuroScope plugin. Follow these step-by-step procedures to quickly resolve problems and get back to normal operations.

## Quick Diagnostic Steps

Before diving into specific issues, run through this quick diagnostic sequence:

<Steps>
  <Step title="Check Basic Status">
    Verify fundamental system state: ``` ✓ EuroScope is running and connected to
    network ✓ BARS plugin appears in loaded plugins list ✓ BARS menu button
    visible on appropriate displays ✓ Internet connection active and stable ```
  </Step>

  <Step title="Verify Configuration">
    Confirm essential settings: ``` ✓ API key entered correctly and
    authenticated ✓ Display types configured to allow BARS drawing ✓ Active
    airports set in EuroScope runways configuration ✓ At least one runway
    claimed for testing ```
  </Step>

  <Step title="Test Basic Functionality">
    Perform simple operational tests: ``` ✓ BARS menu opens and displays options
    ✓ Connection status shows active (not "BARS off") ✓ Runway ownership menu
    displays available runways ✓ Stopbar elements visible on radar display ```
  </Step>
</Steps>

<Info>
  If all quick diagnostic steps pass but you're still experiencing issues,
  proceed to the specific problem sections below.
</Info>

## Connection and Authentication Issues

### Plugin Won't Connect to BARS Server

<Tabs>
  <Tab title="Symptoms">
    - BARS menu button shows "BARS off" permanently - "Connect" option in menu
    has no effect - Connection attempts time out or fail silently - Red status
    indicators in plugin interface
  </Tab>

  <Tab title="Solutions">
    **Step 1: Network Connectivity** 1. **Test internet connection** using web
    browser 2. **Check firewall settings** - ensure EuroScope has network access
    3. **Verify proxy configuration** if applicable 4. **Try different network**
    (mobile hotspot) to isolate network issues **Step 2: Server Status
    Verification** 1. **Visit [stopbars.com](https://stopbars.com)** to verify
    server availability 2. **Check status page** for known server issues 3.
    **Contact support** if server appears down **Step 3: Plugin Reset** 1.
    **Close EuroScope completely** 2. **Restart EuroScope** and reload profile
    3. **Reload BARS plugin** if it doesn't load automatically 4. **Attempt
    connection** after full restart
    <Warning>
      If connection issues persist across multiple sessions and networks,
      contact technical support with your API key and detailed error
      descriptions.
    </Warning>
  </Tab>
</Tabs>

### API Key Authentication Failures

<AccordionGroup>
  <Accordion title="Invalid API Key Error">
    **Symptoms**: Plugin rejects API key with "invalid" or "unauthorized"
    message **Solutions**: 1. **Verify API key accuracy**: - Copy directly from
    BARS website account settings - Ensure no extra spaces or characters - Check
    for case sensitivity issues 2. **Regenerate API key**: - Log into BARS
    website account - Generate new API key - Update plugin with new key 3.
    **Account verification**: - Confirm BARS account is active - Verify account
    has controller privileges - Check for any account restrictions
  </Accordion>

  <Accordion title="Authentication Timeout">
    **Symptoms**: Plugin appears to hang during authentication process
    **Solutions**: 1. **Check network latency**: - Test connection speed to
    stopbars.com - Ensure stable connection without packet loss 2. **Wait for
    timeout completion**: - Allow 30-60 seconds for authentication - Don't
    interrupt process prematurely 3. **Retry authentication**: - Clear API key
    and re-enter - Restart plugin if necessary - Try during different network
    conditions
  </Accordion>
</AccordionGroup>

## Display and Interface Issues

### BARS Menu Button Not Visible

<Steps>
  <Step title="Check Display Type Configuration">
    Verify the plugin is allowed on your current display: 1. **Open EuroScope
    Plugins dialog** 2. **Find BARS in loaded plugins list** 3. **Check "Allowed
    to draw on types" section** 4. **Move current display type** from
    "Forbidden" to "Allowed"
    <Check>
      The BARS menu button should appear immediately after enabling the display
      type.
    </Check>
  </Step>

  <Step title="Verify Display Type Compatibility">
    Ensure you're using an appropriate display for ground operations:
    **Compatible display types**: - SMR (Surface Movement Radar) - Ground radar
    displays - Tower displays - Combined ground/air displays (if configured
    properly) **Incompatible display types**: - Pure airborne radar screens -
    En-route displays - Approach displays without ground elements
  </Step>

  <Step title="Interface Scaling Issues">
    Address potential display scaling problems: 1. **Check Windows display
    scaling** settings 2. **Verify EuroScope interface scaling** configuration
    3. **Look for button in alternative locations** (corners, edges) 4. **Reset
    EuroScope display settings** if necessary
  </Step>
</Steps>

### Stopbars Not Visible or Clickable

<Tabs>
  <Tab title="Missing Stopbar Elements">
    **Troubleshooting invisible stopbars**: 1. **Verify runway ownership**: -
    Ensure you've claimed the runway - Check runway ownership menu shows claimed
    status - Verify you're controlling the correct airport 2. **Check display
    zoom and position**: - Zoom to appropriate level for ground operations -
    Center display on runway areas - Adjust brightness/contrast if needed 3.
    **Confirm airport database coverage**: - Verify airport exists in BARS
    database - Check that specific runways are supported - Contact support for
    database updates if needed
  </Tab>

  <Tab title="Stopbars Not Responding to Clicks">
    **Fixing unresponsive stopbar controls**: 1. **Verify connection status**: -
    Ensure BARS shows connected status - Check for network connectivity issues -
    Attempt reconnection if necessary 2. **Check click precision**: - Click
    directly on stopbar elements, not nearby - Understand stopbars may not align
    perfectly with EuroScope graphics - Try different zoom levels for easier
    clicking 3. **Mouse and interface settings**: - Verify mouse sensitivity and
    click responsiveness - Check for conflicting EuroScope mouse functions -
    Ensure no other software is intercepting clicks
  </Tab>
</Tabs>

## Operational Issues

### Runway Ownership Problems

<AccordionGroup>
  <Accordion title="No Runways Available to Claim">
    **Symptoms**: Runway ownership menu appears empty or shows no claimable
    runways **Root Causes and Solutions**: 1. **Airport configuration**: - Set
    active airports in EuroScope runways popup - Ensure airport codes are
    correct (ICAO format) - Verify spelling and case sensitivity 2. **Database
    coverage**: - Confirm airport exists in BARS database - Check if specific
    runways are supported - Request airport addition if needed 3. **Network
    connectivity**: - Verify connection to BARS server - Check for
    authentication issues - Ensure real-time data synchronization
  </Accordion>

  <Accordion title="Cannot Unclaim Runways">
    **Symptoms**: Runways remain claimed despite attempts to release them
    **Solutions**: 1. **Force release procedures**: - Click runway multiple
    times in ownership menu - Close and reopen runway ownership dialog - Restart
    plugin connection if necessary 2. **Session management**: - Ensure proper
    session termination when logging off - Verify other controllers aren't
    claiming same runways - Use manual release commands if available
  </Accordion>
</AccordionGroup>

### Pilot Connectivity Indicators

<Steps>
  <Step title="Missing Green Dots on BARS-Equipped Aircraft">
    Investigate indicator display issues: 1. **Verify pilot is actually using
    BARS**: - Confirm with pilot via voice communication - Check pilot client
    compatibility - Understand not all pilots use BARS 2. **Check indicator
    settings**: - Ensure indicators are enabled in plugin options - Verify
    display brightness and visibility - Check for indicator position conflicts
  </Step>

  <Step title="False Positive Indicators">
    Address incorrect green dot displays: 1. **Verify actual pilot equipment**:
    - Confirm BARS usage through pilot communication - Check for pilot client
    version compatibility - Report persistent false positives to support 2.
    **Plugin data synchronization**: - Restart plugin to refresh pilot status -
    Verify server connection integrity - Check for plugin version updates
  </Step>
</Steps>

## Performance and Stability Issues

### Plugin Crashes or Freezes

<Tabs>
  <Tab title="Immediate Recovery">
    **Quick steps to restore functionality**: 1. **Close problematic dialogs**
    or menus if possible 2. **Save EuroScope session** to prevent data loss 3.
    **Restart EuroScope** if plugin becomes completely unresponsive 4. **Reload
    plugin** and reconfigure as necessary **Prevention measures**: - Avoid rapid
    clicking or menu navigation - Allow sufficient time for network operations -
    Close unused plugin dialogs to reduce memory usage
  </Tab>

  <Tab title="Stability Improvements">
    **Long-term stability enhancements**: 1. **Update software**: - Install
    latest BARS plugin version - Update EuroScope to current release - Ensure
    Windows is current with updates 2. **System optimization**: - Close
    unnecessary background applications - Ensure adequate system memory - Check
    for malware or system conflicts 3. **Configuration optimization**: - Reduce
    number of simultaneous plugin operations - Optimize network settings for
    stability - Configure appropriate timeout values
  </Tab>
</Tabs>

### Slow Response Times

<AccordionGroup>
  <Accordion title="Network Latency Issues">
    **Symptoms**: Delayed response to stopbar commands, slow menu operations
    **Solutions**: 1. **Network optimization**: - Use wired connection instead
    of WiFi when possible - Check for network congestion during peak hours -
    Consider upgrading internet connection speed 2. **Server communication**: -
    Verify connection to optimal BARS server - Check for regional server
    alternatives - Report persistent latency issues to support
  </Accordion>

  <Accordion title="System Performance">
    **Symptoms**: General sluggishness, delayed interface responses
    **Solutions**: 1. **System resources**: - Close unnecessary applications -
    Check available memory and CPU usage - Restart computer if running for
    extended periods 2. **EuroScope optimization**: - Reduce display complexity
    when possible - Limit number of simultaneous radar windows - Optimize
    EuroScope settings for performance
  </Accordion>
</AccordionGroup>

## Error Messages and Codes

### Common Error Messages

<Tabs>
  <Tab title="Authentication Errors">
    **"Invalid API Key"** - Verify key copied correctly from account settings -
    Check for extra spaces or hidden characters - Regenerate new API key if
    problem persists **"Authentication Failed"** - Confirm internet connectivity
    - Check BARS server status - Verify account is active and in good standing
    **"Account Suspended"** - Contact BARS support immediately - Review account
    terms and conditions - Resolve any outstanding account issues
  </Tab>

  <Tab title="Connection Errors">
    **"Server Unavailable"** - Check BARS server status page - Verify internet
    connection - Wait and retry connection **"Connection Timeout"** - Check
    network stability - Verify firewall settings - Try connection during
    off-peak hours **"Protocol Error"** - Update plugin to latest version -
    Restart EuroScope completely - Contact support if error persists
  </Tab>

  <Tab title="Operational Errors">
    **"Runway Not Available"** - Verify runway exists in BARS database - Check
    airport code spelling - Ensure runway is not claimed by another controller
    **"Invalid Operation"** - Verify you have ownership of the runway - Check
    connection status - Ensure stopbar exists at clicked location **"Permission
    Denied"** - Verify controller authorization level - Check account
    permissions - Ensure proper VATSIM rating if required
  </Tab>
</Tabs>

## Getting Additional Help

### Information to Collect Before Contacting Support

When reporting issues, provide the following information for faster resolution:

<Steps>
  <Step title="System Information">
    Gather technical details: ``` - EuroScope version and build number - BARS
    plugin version - Windows version and build - Network connection type and
    speed - Firewall and antivirus software in use ```
  </Step>

  <Step title="Error Details">
    Document the specific problem: ``` - Exact error messages (screenshots
    preferred) - Steps to reproduce the issue - Frequency of occurrence -
    Workarounds that have been attempted - Impact on operations ```
  </Step>

  <Step title="Account Information">
    Provide relevant account details: ``` - BARS account username (not password)
    - Affected airports and runways - Time and date of issues - VATSIM CID if
    applicable ```
  </Step>
</Steps>

### Emergency Procedures

<Warning>
  If BARS fails during active control operations, immediately fall back to
  standard ground control procedures using voice communication and traditional
  holding point references.
</Warning>

**Emergency fallback steps**:

1. **Announce system failure** to affected pilots
2. **Revert to standard phraseology** and procedures
3. **Increase verbal communication** frequency
4. **Use traditional holding points** for aircraft positioning
5. **Document incident** for later troubleshooting
6. **Contact technical support** after operations conclude

<Note>
  Safety always takes precedence over technology. Never compromise operational
  safety while troubleshooting technical issues.
</Note>
