# Installing BARS Client

Get up and running with BARS Client in just a few minutes. This guide covers downloading, installing, and initial configuration of the BARS Client for Microsoft Flight Simulator.

## Download BARS Client

<Steps>
<Step title="Visit the Official Website">
Navigate to [stopbars.com](https://stopbars.com) to access the official BARS Client download page.

<Note>
Always download BARS Client from the official website to ensure you get the latest, secure version.
</Note>
</Step>

<Step title="Download the Installer">
Click the download button to get the latest BARS Client installer for Windows.

The installer file will be named something like `BARSClient-Setup-v1.x.x.exe`.

</Step>

<Step title="Verify the Download">
Once downloaded, verify the file integrity (optional but recommended for security).

<Tip>
The download page includes file checksums that you can use to verify your download completed successfully.
</Tip>
</Step>
</Steps>

## Installation Process

<Steps>
<Step title="Run the Installer">
Right-click the downloaded installer and select "Run as administrator" to ensure proper installation permissions.

<Warning>
You may see a Windows SmartScreen warning for new software. Click "More info" and then "Run anyway" to proceed with the installation.
</Warning>
</Step>

<Step title="Follow Installation Wizard">
The installation wizard will guide you through the setup process:

1. **Accept the license agreement**
2. **Choose installation directory** (default location is recommended)
3. **Select additional components** if prompted
4. **Complete the installation**

<Frame>
<img src="/images/BARS_client_install.png" alt="BARS Client installation window showing the setup process" />
</Frame>
</Step>

<Step title="Verify Installation">
After installation completes, verify BARS Client is properly installed:

1. Search for "BARS Client" in the Windows Start menu
2. Launch the application
3. The client should minimize to the system tray upon startup

<Check>
If you can see the BARS Client icon in your system tray, the installation was successful.
</Check>
</Step>
</Steps>

## Initial Configuration

<Steps>
<Step title="Create BARS Account">
Before you can use BARS Client, you need to create an account:

1. Visit [stopbars.com](https://stopbars.com)
2. Click "Sign Up" to create a new account
3. Complete the registration process
4. Verify your email address if required

<Note>
Your BARS account provides access to the stopbar network and allows controllers to interact with your client.
</Note>
</Step>

<Step title="Obtain Your API Token">
Your API token is required for authentication:

1. Log into your BARS account
2. Navigate to "Account Settings"
3. Click "Show Token" to reveal your unique API token
4. Copy this token for use in the client

<Warning>
  Never share your API token with anyone. This token is unique to your account
  and provides access to BARS services.
</Warning>

<Frame>
<img src="/images/BARS_token.png" alt="BARS website showing the API token in account settings" />
</Frame>
</Step>

<Step title="Configure BARS Client">
Open BARS Client from the system tray and configure your settings:

1. **Right-click the BARS Client icon** in the system tray
2. **Select "Show"** to open the client window
3. **Paste your API token** into the "API Token" field
4. **Select your preferred stopbar behavior** (realistic or enhanced)
5. **Configure scenery settings** if using third-party airports

<Frame>
<img src="/images/BARS_client_connection.png" alt="BARS Client interface showing connection status and settings" />
</Frame>
</Step>
</Steps>

## Stopbar Behavior Configuration

Choose how stopbars appear in your simulator based on your preference:

<Tabs>
<Tab title="Realistic Mode">
**Recommended for maximum realism**

- **Active stopbars**: Illuminated red lights
- **Inactive stopbars**: Unilluminated (dark)
- **Behavior**: Matches real-world airport operations

This mode provides the most authentic experience, mimicking how stopbars actually operate at real airports.

</Tab>

<Tab title="Enhanced Mode">
**Recommended for better visibility**

- **Active stopbars**: Illuminated red lights
- **Inactive stopbars**: Illuminated green lights
- **Behavior**: Enhanced visual feedback

This mode uses green lights to clearly indicate when you're cleared to proceed, which can be helpful in low-visibility conditions or for training purposes.

</Tab>
</Tabs>

## Third-Party Scenery Setup

If you use payware airport scenery, configure BARS for optimal compatibility:

<Steps>
<Step title="Access Scenery Settings">
In the BARS Client window, navigate to the "Scenery" section.
</Step>

<Step title="Select Your Airport">
  Find or search for your airport in the compatible airports list.
</Step>

<Step title="Choose Scenery Type">
Click the dropdown menu for your airport and select your specific payware scenery from the available options.

<Tip>
Popular scenery developers like FlyTampa, FSDT, and Aerosoft often have specific BARS configurations available.
</Tip>
</Step>

<Step title="Apply Changes">
Click "Restart to Apply Changes" at the bottom of the client window. This will restart the BARS client with your new scenery configuration.

<Frame>
<img src="/images/BARS_client_scenery.png" alt="BARS Client scenery configuration showing airport and scenery selection" />
</Frame>
</Step>
</Steps>

## Verification and Testing

<Steps>
<Step title="Launch Microsoft Flight Simulator">
Start MSFS and spawn at a [BARS-compatible airport](/pilot-client/airports).
</Step>

<Step title="Check Client Status">
Verify BARS Client is working properly:

1. **Check system tray**: BARS Client icon should be visible
2. **Open client window**: Right-click icon and select "Show"
3. **Verify connection**: Status should show "Connected"
4. **Check airport detection**: Client should display your current airport

<Check>
If the client shows your current airport and "Connected" status, you're ready to use BARS.
</Check>
</Step>

<Step title="Wait for Stopbar Loading">
When a controller connects to the network, stopbars will automatically load:

- **Loading time**: Maximum 2 minutes after controller connection
- **Visual indication**: Stopbar lights will appear at holding points
- **Client confirmation**: Stopbar count will be displayed in the client

<Note>
Stopbars only load when a BARS-equipped controller is actively controlling your airport. Without a controller present, no stopbars will be visible.
</Note>
</Step>
</Steps>

## Post-Installation Settings

### Windows Firewall

BARS Client may require firewall permissions to connect to the internet:

<Tabs>
<Tab title="Automatic Configuration">
Most users won't need to manually configure firewall settings, as Windows will prompt you to allow BARS Client when it first attempts to connect.

When prompted, select "Allow access" to permit BARS Client to communicate with the servers.

</Tab>

<Tab title="Manual Configuration">
If you need to manually configure your firewall:

1. Open Windows Defender Firewall
2. Click "Allow an app or feature through Windows Defender Firewall"
3. Click "Change Settings" and then "Allow another app"
4. Browse to your BARS Client installation directory
5. Select `BARSClient.exe` and click "Add"
6. Ensure both "Private" and "Public" are checked

</Tab>
</Tabs>

### Antivirus Considerations

Some antivirus software may flag BARS Client as potentially unwanted software:

<Warning>
  If your antivirus software quarantines or blocks BARS Client, you'll need to
  add it to your exclusion list or whitelist. This is common with new software
  that modifies simulator behavior.
</Warning>

**Common solutions**:

- Add BARS Client installation folder to antivirus exclusions
- Add `BARSClient.exe` to your antivirus whitelist
- Temporarily disable real-time protection during installation

## Troubleshooting Installation Issues

<AccordionGroup>
<Accordion title="Installation Fails or Won't Start">
**Common causes and solutions**:

- **Insufficient permissions**: Run installer as administrator
- **Antivirus interference**: Temporarily disable antivirus during installation
- **Previous installation conflicts**: Uninstall any previous BARS versions first
- **Windows compatibility**: Ensure you're running Windows 10 or 11

If issues persist, try downloading a fresh copy of the installer.

</Accordion>

<Accordion title="Cannot Find BARS Client After Installation">
**Check these locations**:

- **System tray**: Look for the BARS icon in the notification area
- **Start menu**: Search for "BARS Client"
- **Installation directory**: Default is `C:\Program Files\BARS Client\`
- **Task Manager**: Check if `BARSClient.exe` is running

If not found, try reinstalling with administrator privileges.

</Accordion>

<Accordion title="Token Authentication Fails">
**Verification steps**:

- **Check token format**: Ensure you copied the complete token
- **Account status**: Verify your BARS account is active
- **Network connectivity**: Ensure internet connection is stable
- **Firewall blocking**: Check if firewall is blocking BARS Client

Try logging out and back into your BARS account to refresh your token.

</Accordion>
</AccordionGroup>
