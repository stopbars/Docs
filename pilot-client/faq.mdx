# Frequently Asked Questions

Find quick answers to the most common questions about BARS Client, from basic operation to advanced troubleshooting scenarios.

## General Questions

### What is BARS and how does it work?

BARS (Runway Approach Recognition System) is a plugin system that brings realistic stopbar operations to Microsoft Flight Simulator. It creates a shared environment where controllers can manage ground traffic using illuminated stopbars, just like in real airports.

<AccordionGroup>
<Accordion title="How does BARS enhance realism?">
**Real-world accuracy**: BARS simulates actual airport stopbar systems used at major international airports worldwide.

**Controller integration**: When BARS-equipped controllers are online, they can manage your ground movements using precise stopbar control instead of relying solely on voice instructions.

**Safety simulation**: Experience the same safety systems that prevent runway incursions in real aviation operations.

**Traffic management**: See how ground traffic is efficiently managed through coordinated stopbar operations.

</Accordion>

<Accordion title="Do I need special hardware?">
**No special hardware required**: BARS works with any standard Microsoft Flight Simulator setup.

**Standard system requirements**:

- Windows 10/11 (64-bit)
- Microsoft Flight Simulator 2020
- Stable internet connection
- Standard gaming computer specifications

**Optional enhancements**:

- High-resolution monitor for better stopbar visibility
- Good graphics card for enhanced lighting effects
- Fast internet connection for optimal responsiveness
</Accordion>
</AccordionGroup>

### Is BARS free to use?

**Yes, BARS Client is completely free** for all pilots. There are no subscription fees, premium features, or hidden costs.

**What's included**:

- Full BARS Client software
- Access to all supported airports
- Regular updates and new features
- Community support and documentation
- Integration with compatible ATC clients

<Note>
  BARS is a community-driven project supported by donations and volunteer
  development. Consider supporting the project if you find it valuable.
</Note>

### Can I use BARS without being on VATSIM or IVAO?

**Yes, BARS can work outside of virtual networks**, but functionality depends on having BARS-equipped controllers:

<Tabs>
<Tab title="With Networks (VATSIM/IVAO)">
**Full functionality**: 
- Regular BARS-equipped controllers
- Busy traffic scenarios
- Realistic operations
- Community support

**Recommended for**: Pilots wanting regular BARS experience with active ATC

</Tab>

<Tab title="Without Networks">
**Limited functionality**:
- Requires coordinating with other BARS users
- Manual setup of controller sessions
- Less frequent opportunities to use BARS

**Recommended for**: Testing, training, or special events

</Tab>

<Tab title="Offline/Solo">
**No BARS functionality**: Stopbars require an active controller to operate. Solo flights won't show any stopbar activity.

**Alternative**: Use BARS time to learn airport layouts and practice procedures for when controllers are available.

</Tab>
</Tabs>

## Installation and Setup

### Why won't BARS Client install on my computer?

<AccordionGroup>
<Accordion title="Common installation blockers">
**Antivirus interference**: Many antivirus programs flag BARS as potentially unwanted software.

_Solution_: Add BARS to your antivirus exclusions or temporarily disable protection during installation.

**Insufficient permissions**: Installation requires administrator privileges.

_Solution_: Right-click the installer and select "Run as administrator."

**Corrupted download**: Download may be incomplete or corrupted.

_Solution_: Download a fresh copy from the official website and verify checksums if provided.

**Windows SmartScreen**: Windows may block unrecognized software.

_Solution_: Click "More info" on the warning and select "Run anyway."

</Accordion>

<Accordion title="System compatibility issues">
**Outdated Windows version**: BARS requires Windows 10 or 11.

_Solution_: Update to a supported Windows version.

**Missing dependencies**: BARS requires .NET Framework and Visual C++ Redistributables.

_Solution_: Install latest versions from Microsoft's website.

**Architecture mismatch**: BARS requires 64-bit Windows.

_Solution_: Verify you're running 64-bit Windows in System Properties.

**Disk space**: Insufficient space can cause installation failures.

_Solution_: Free up at least 1GB of space on your system drive.

</Accordion>
</AccordionGroup>

### My API token isn't working. What should I do?

**Step-by-step token troubleshooting**:

<Steps>
<Step title="Verify Token Accuracy">
1. Log into your BARS account on the website
2. Navigate to Account Settings
3. Click "Show Token" to reveal your current token
4. Copy the entire token (usually a long alphanumeric string)
5. Paste directly into BARS Client without any modifications

<Warning>
Ensure you copy the complete token without extra spaces, line breaks, or partial copying.
</Warning>
</Step>

<Step title="Check Account Status">
  - **Email verification**: Ensure your account email is verified - **Account
  suspension**: Check for any suspension notices in your email - **Token
  expiration**: Some tokens may expire - try regenerating a new one - **Account
  deletion**: Verify your account still exists and is active
</Step>

<Step title="Test Connection">
- **Internet connectivity**: Verify you can access other websites
- **Firewall blocking**: Ensure BARS Client can access the internet
- **Regional restrictions**: Some regions may have limited access
- **Server issues**: Check BARS Discord for server status updates
</Step>
</Steps>

### How do I configure BARS for third-party scenery?

**Automatic detection works for most popular sceneries**, but manual configuration may be needed:

<Steps>
<Step title="Identify Your Scenery">
Determine which airport addon you're using:
- Check your MSFS content manager
- Look for scenery developer (FlyTampa, FSDT, Aerosoft, etc.)
- Note the specific airport (KJFK, EGLL, EDDF, etc.)
</Step>

<Step title="Configure in BARS Client">
1. Open BARS Client from system tray
2. Navigate to "Scenery" section
3. Search for your airport by ICAO code
4. Select your scenery developer from dropdown
5. Click "Restart to Apply Changes"

<Check>
After restart, BARS should properly detect and configure stopbars for your scenery.
</Check>
</Step>

<Step title="Verify Configuration">
- Spawn at the airport in MSFS
- Check that BARS Client detects the correct airport
- Look for properly positioned stopbars when a controller is online
- Report any positioning issues to the community
</Step>
</Steps>

## Operational Questions

### Why don't I see any stopbars at the airport?

**Stopbars only appear when specific conditions are met**:

<AccordionGroup>
<Accordion title="Controller Requirements">
**BARS-equipped controller must be online**: Not all controllers have BARS installed or enabled.

**Active control**: Controller must be actively controlling your airport with BARS functionality enabled.

**Network connectivity**: Both you and the controller need stable internet connections.

**Initialization time**: Allow up to 2 minutes for stopbars to load after controller connection.

_How to check_: Ask the controller if they're using BARS, or check their controller info for BARS compatibility.

</Accordion>

<Accordion title="Airport Compatibility">
**Supported airport**: Your airport must be on the [BARS compatibility list](/pilot-client/airports).

**Proper positioning**: You must be on the ground at taxiways or gates, not airborne or on runways.

**Scenery configuration**: Third-party scenery must be properly configured in BARS Client.

_How to verify_: Check the compatibility list and ensure your scenery is configured correctly.

</Accordion>

<Accordion title="Technical Issues">
**Connection problems**: BARS Client must show "Connected" status.

**Authentication failures**: Your API token must be valid and properly entered.

**Client malfunctions**: BARS Client may need restarting or reinstalling.

_How to troubleshoot_: Check connection status, verify token, and restart the client if needed.

</Accordion>
</AccordionGroup>

### What do different stopbar colors mean?

<Tabs>
<Tab title="Realistic Mode">
**Matches real-world airports**:

- **Red lights**: STOP - Do not proceed past this point
- **No lights (dark)**: PROCEED - Safe to continue taxi
- **Flashing red**: Emergency stop (rare, used in emergencies)

This mode provides the most authentic experience but may be harder to see in bright conditions.

</Tab>

<Tab title="Enhanced Mode">
**Improved visibility**:

- **Red lights**: STOP - Hold your position
- **Green lights**: PROCEED - Cleared to continue
- **No lights**: System inactive or malfunctioning

This mode offers clearer visual feedback, especially helpful for training or low-visibility conditions.

</Tab>
</Tabs>

**Mode selection**: Choose your preferred mode in BARS Client settings based on your visibility needs and realism preferences.

### Should I follow stopbars or radio instructions when they conflict?

**Priority order for conflicting instructions**:

<Steps>
<Step title="Immediate Safety">
**Stop immediately** if there's any potential safety conflict, regardless of other instructions.
</Step>

<Step title="Stopbar Priority">
  **Follow stopbar indications** - they represent the controller's most current
  intent and override previous radio clearances.
</Step>

<Step title="Query Controller">
**Ask for clarification** if the conflict persists:

_Example_: "Ground, [callsign] has conflicting clearance and stopbar indication at Alpha intersection."

</Step>

<Step title="Wait for Resolution">
**Don't proceed** until the conflict is clearly resolved through radio communication.
</Step>
</Steps>

<Warning>
  Never cross a red stopbar based on a previous radio clearance. The stopbar
  represents real-time control and takes precedence.
</Warning>

## Technical Questions

### Does BARS affect my simulator performance?

**Minimal performance impact** for most users:

<Tabs>
<Tab title="Frame Rate">
**Typical impact**: 0-2 FPS reduction under normal conditions

**Factors affecting performance**:

- Number of active stopbars
- Graphics settings (especially lighting and bloom)
- System hardware specifications
- Other running applications

**Optimization tips**:

- Update graphics drivers
- Adjust MSFS lighting settings if needed
- Close unnecessary background applications
- Enable hardware acceleration if available
</Tab>

<Tab title="Memory Usage">
**RAM usage**: BARS Client typically uses 50-100MB of system memory

**Storage space**: Installation requires less than 100MB disk space

**Network bandwidth**: Minimal usage (similar to web browsing)

**System resources**: Negligible CPU usage during normal operation

</Tab>
</Tabs>

## Troubleshooting

### BARS worked before but stopped working. What changed?

**Common causes of sudden BARS failures**:

<AccordionGroup>
<Accordion title="Recent Changes">
**Software updates**: Windows updates, MSFS updates, or graphics driver updates may affect BARS

_Solution_: Restart BARS Client and computer, check for BARS updates

**New addon installations**: Recently installed scenery or aircraft addons might conflict

_Solution_: Test BARS with default scenery and aircraft to isolate issues

**Network changes**: Router settings, ISP changes, or firewall updates

_Solution_: Check firewall settings and test internet connectivity

**Account issues**: Token expiration or account status changes

_Solution_: Verify account status and regenerate API token if needed

</Accordion>

<Accordion title="System Issues">
**Hard drive corruption**: File system errors can affect BARS operation

_Solution_: Run disk check utilities and scan for system file corruption

**Registry problems**: Windows registry corruption can cause application failures

_Solution_: Use system restore or registry cleaning tools

**Memory issues**: RAM problems or system instability

_Solution_: Run memory diagnostic tools and check system stability

**Malware**: Virus or malware infection affecting system operation

_Solution_: Run comprehensive antivirus scan and malware removal tools

</Accordion>
</AccordionGroup>

### How do I completely uninstall and reinstall BARS?

**Complete removal and fresh installation**:

<Steps>
<Step title="Uninstall BARS Client">
1. Close BARS Client completely (check system tray)
2. Open Windows Settings > Apps & Features
3. Find "BARS Client" and click Uninstall
4. Follow the uninstallation wizard
5. Restart your computer
</Step>

<Step title="Clean Remaining Files">
  1. Delete BARS installation folder (usually in Program Files) 2. Remove user
  data: `%localappdata%\BARS\` 3. Clear temporary files and registry entries if
  needed 4. Empty recycle bin
</Step>

<Step title="Fresh Installation">
  1. Download latest BARS Client from official website 2. Run installer as
  Administrator 3. Follow installation wizard with default settings 4. Configure
  with your API token 5. Set up scenery preferences
</Step>

<Step title="Verify Installation">
- Check system tray for BARS icon
- Verify connection status shows "Connected"
- Test at a compatible airport with controller online
- Confirm stopbars appear and function properly
</Step>
</Steps>
