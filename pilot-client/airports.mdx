# BARS-Compatible Airports

BARS Client supports stopbar operations at airports where ground infrastructure has been modeled and configured. This page provides a comprehensive list of supported airports, including compatibility with popular third-party scenery packages.

<Note>
  This list is regularly updated as new airports are added to the BARS network.
  If you don't see your favorite airport listed, check back regularly or request
  support through our community channels.
</Note>

## How Airport Compatibility Works

BARS requires specific ground infrastructure modeling to function properly:

<Steps>
<Step title="Stopbar Infrastructure">
Airports must have stopbar positions defined for:
- **Runway holding points**: Primary safety stopbars at runway entrances
- **Taxiway intersections**: Traffic flow management points
- **Critical areas**: Sensitive navigation aid protection zones
</Step>

<Step title="Scenery Integration">
  Stopbars are integrated with: - **Default MSFS scenery**: Built-in
  compatibility for major airports - **Third-party scenery**: Specific
  configurations for popular payware - **Custom configurations**:
  Community-developed stopbar layouts
</Step>

<Step title="Controller Support">
Full functionality requires:
- **BARS-equipped controllers**: Using compatible ATC clients
- **Network presence**: Controllers actively using BARS plugins
- **Proper training**: Controllers familiar with stopbar operations
</Step>
</Steps>

### Configuration Requirements

<Steps>
<Step title="Automatic Detection">
Most premium sceneries are automatically detected by BARS Client:

- **Popular developers**: Pre-configured compatibility
- **Common airports**: Built-in support profiles
- **Automatic switching**: Client detects active scenery
</Step>

<Step title="Manual Configuration">
For some sceneries, manual selection may be required:

1. **Open BARS Client** scenery settings
2. **Find your airport** in the compatibility list
3. **Select scenery developer** from dropdown
4. **Restart client** to apply changes

</Step>

<Step title="Custom Profiles">
Advanced users can create custom profiles:

- **Community contributions**: User-generated configurations
- **Special requirements**: Unique airport layouts
- **Beta testing**: New scenery compatibility
</Step>
</Steps>

## Reporting Compatibility Issues

<AccordionGroup>
<Accordion title="Stopbars Not Appearing">
**Troubleshooting steps**:

1. **Verify airport support**: Check this compatibility list
2. **Confirm controller presence**: Ensure BARS-equipped ATC is online
3. **Check scenery selection**: Verify correct third-party scenery is selected
4. **Wait for loading**: Allow up to 2 minutes for stopbar initialization

**Report if**: Stopbars still don't appear after troubleshooting

</Accordion>

<Accordion title="Incorrect Stopbar Positions">
**Common issues**:

- Stopbars appearing in wrong locations
- Missing stopbars at critical intersections
- Stopbars interfering with aircraft movement

**Information to provide**:

- Airport ICAO code
- Scenery developer and version
- Screenshots of issues
- Aircraft position coordinates
</Accordion>

<Accordion title="Performance Issues">
**Performance-related problems**:

- Frame rate drops when stopbars load
- Client crashes at specific airports
- Excessive memory usage

**Solutions to try**:

- Update graphics drivers
- Reduce MSFS graphics settings
- Check system resources
- Report persistent issues
</Accordion>
</AccordionGroup>
